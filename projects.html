1. Simple dictionary using a linked list in C

What is a Linked List??

The Train of Data üöÇ

Imagine you‚Äôre at a train station, and instead of one long train, there are train cars connected one after another. Each car has two things:

  • Passengers (data): The information stored in that train car.
  • A connection to the next train car: A link that tells where the next car is.

But here‚Äôs the twist: Unlike a normal train where you can easily go to the previous car, a basic Linked List only knows where the next car is! If you want to reach the last car, you have to start from the front and move one by one.

Types of Linked Lists:

  • Singly Linked List: The train moves forward only (each car only links to the next one).
  • Doubly Linked List: The train can move forward and backward (each car links to both the next and previous ones).
  • Circular Linked List: The train is infinite! The last car links back to the first one.

My project

I created a simple dictionary using a linked list in C to store information from a dataset about the City of Melbourne. This dictionary allowed users to search for footpath information using attributes from the dataset as keys.

To build the dictionary, I read data from a file and inserted each footpath record as a node in a linked list. Additionally, I implemented a method to search for a key within the list and output any matching records. Since keys were not guaranteed to be unique, multiple records could be retrieved for a single search.

2. PR quadtree

What is a PR Quadtree??

The City Map of Data üó∫Ô∏è

Imagine you‚Äôre looking at a map of a city, and you want to organize it efficiently. Instead of storing every single location individually, you divide the map into four regions. Each of these regions can be further split into four smaller regions, just like zooming into a digital map.

This is how a Point Region (PR) Quadtree works! It breaks down space into four quadrants whenever there are multiple points in the same area, helping efficiently store and search spatial data.

How It Works:

  • Each node represents a region. If there are too many points in one region, it splits into four smaller regions.
  • Each region can contain at most one point. If a new point is added in an occupied region, it divides further.
  • The tree grows dynamically. Only areas with data are divided, keeping memory usage efficient.

Applications of PR Quadtrees:

  • üó∫Ô∏è Geographical Mapping: Used in GPS and mapping applications.
  • üéÆ Game Development: Helps manage large game worlds efficiently.
  • üîç Spatial Searching: Quickly finds objects in 2D space.
  • üõ∞Ô∏è Image Processing: Used in image compression and terrain modeling.

My project

I implemented a PR quadtree to store and search for data points efficiently, this was an extension to the previous city of Melbourne data assignment. The implementation supported inserting data points, searching for individual points, and retrieving all points within a query rectangle. For insertion, if the tree was empty and the point was within bounds, it became the root. Otherwise, I recursively traversed the tree to find the correct position. If a black leaf already contained a point, I split it into four children, continuing until the new and existing points occupied separate black leaf nodes.

For searching, I traversed the tree, selecting child nodes based on the point's location. If found, the associated information was returned; otherwise, the search indicated the point was not stored.

For range queries, I recursively checked which child nodes overlapped with the query rectangle, retrieving points from all black leaf nodes within the specified area.

3. Solver for the Flow Free game

What is Dijkstra‚Äôs Algorithm??

The GPS of Shortest Paths üó∫Ô∏è

Imagine you‚Äôre in a city with many roads, and you need to find the shortest route from your house to a friend‚Äôs place. Instead of randomly guessing which way to go, you use a GPS that calculates the quickest route based on distance. That‚Äôs exactly what Dijkstra‚Äôs Algorithm does in the world of graphs.

This algorithm finds the shortest path from one starting point to all other points in a network, just like a GPS finds the fastest way to multiple locations.

How It Works:

  • Start from a chosen point (source) and give it a distance of zero.
  • All other points start with an infinite distance (since we don‚Äôt know how far they are yet).
  • Visit the nearest unvisited point and update distances to its neighbors if a shorter path is found.
  • Repeat the process until all points have the shortest possible distance calculated.

Applications of Dijkstra‚Äôs Algorithm:

  • üöó Finding the shortest driving routes in Google Maps and GPS systems.
  • üåç Network routing in the Internet to send data efficiently.
  • üéÆ Pathfinding for game AI to navigate maps.
  • üí° Planning electricity and water distribution in cities.

My project

In this programming assignment, I built a solver using Djikstra's Algorithm for the Flow Free game, a puzzle dating back to 1897 and popularized in Japan as Numberlink. The game involves connecting matching pairs of dots on a grid without overlapping paths. The assignment code was adapted from the open-source terminal version by mzucker.
